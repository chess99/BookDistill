# Design: 超长书籍提炼策略

Generated by /office-hours + /brainstorming on 2026-04-06
Branch: main
Status: APPROVED

## 背景

当前 BookDistill 对超长书籍（如社会心理学 2438k chars ≈ 610k tokens）会直接失败，因为 MiniMax-M2.7 上下文窗口约 128k tokens。需要两条并行路径解决这个问题，结果输出到仓库下供 review。

## 路径 A：Claude via mc proxy（openai_compatible）

### 原理

美团内部 mc proxy（`https://mcli.sankuai.com`）暴露 OpenAI-compatible 接口，背后是 Claude API。在 config 里加一个 `openai_compatible` provider，模型用 `claude-sonnet-4-5`（200k token context ≈ 800k chars）或 `claude-opus-4-5`。

适用范围：claude-sonnet-4-6 / claude-opus-4-6 均有 1M token 上下文（≈ 4M chars），社会心理学 610k tokens 完全能装下。路径 A 可以直接解决所有当前的超长书问题。

### 配置变更

在 `cli/config.json` 的 `providers` 里加：
```json
"claude": {
  "type": "openai_compatible",
  "apiKey": "b5b927e3b5284d2f8e10de83ea624d9e",
  "baseUrl": "https://mcli.sankuai.com/api/v1.0/gpt"
}
```

模型：`claude-sonnet-4-6`（默认，1M context）或 `claude-opus-4-6`（更贵但质量更高，1M context）

### 实现

无需改代码。`generateSummary()` 的 `openai_compatible` 分支已经 work，`baseUrl` 会被拼接为 `https://mcli.sankuai.com/api/v1.0/gpt/v1/chat/completions`。

需要确认 mc proxy 的实际端点路径（可能是 `/api/v1.0/gpt/chat/completions` 而不是 `/api/v1.0/gpt/v1/chat/completions`）。

### 输出

运行后输出到 `/tmp/social-psych-claude-path-a.md`（或实际能跑的书）

## 路径 B：层级提炼（Hierarchical Distillation）

### 原理

把超长文本切成 N 块，每块独立提炼出"章节摘要"，最后一个合并 pass 综合所有摘要生成最终结果。全程用 MiniMax-M2.7，不依赖大窗口模型。

```
原文（2438k chars）
  ↓ 切割（每块 ~80k chars，留 buffer）
块1 → 中间摘要1（~3k chars）
块2 → 中间摘要2（~3k chars）
...
块N → 中间摘要N（~3k chars）
  ↓ 合并 pass
最终提炼（综合 N 个摘要）
```

### 切割策略

- 目标块大小：80k chars（MiniMax 128k token ≈ 512k chars，留足 prompt + 输出空间）
- 切割点：尽量在段落边界（`\n\n`）而不是句子中间
- 最小块大小：10k chars（避免碎片化）

### Prompt 设计

**中间 pass（每块）：**
```
你正在处理《{title}》第 {n}/{total} 部分的内容。
请提炼这一部分的核心观点、关键概念和重要论据。
格式：要点列表 + 关键引用（如有）。
不需要完整结构，只需忠实记录这部分的核心内容。
```

**合并 pass：**
```
以下是《{title}》各部分的提炼摘要，共 {N} 部分。
请综合这些摘要，生成一份完整的书籍提炼，要求：
- 识别贯穿全书的核心主题和论点
- 整合各部分的关键概念
- 去除重复，突出洞察
- 按主题（而非按部分顺序）组织内容
```

### 实现位置

新建 `src/services/hierarchicalDistill.ts`：
```typescript
export async function distillLargeText(
  text: string,
  title: string,
  author: string,
  language: string,
  generateFn: (text: string, title: string, author: string) => Promise<string>,
  options?: { chunkSize?: number }
): Promise<string>
```

在 `cli/distill.ts` 的 Step 5 前检测文本长度，超过阈值（200k chars）自动走层级路径。

### 阈值

- `LARGE_BOOK_THRESHOLD = 200_000` chars（约 50k tokens，MiniMax 能单次处理的安全上限）
- 超过此阈值自动切换层级模式，打印提示：`Book too large for single pass (Xk chars), using hierarchical distillation...`

### 输出

运行后输出到 `/tmp/social-psych-hierarchical-path-b.md`

## 实现计划

### Step 1：验证路径 A 配置

1. 在 `cli/config.json` 加 `claude` provider
2. 测试端点是否正确（先用小书验证）
3. 用社会心理学 PDF 测试（如果 610k tokens 超出 claude-sonnet-4-5 的 200k 限制，换一本中等大的书）

### Step 2：实现路径 B

1. 新建 `src/services/hierarchicalDistill.ts`
2. 实现 `splitText()`（按段落边界切割）
3. 实现 `distillLargeText()`（中间 pass + 合并 pass）
4. 在 `cli/distill.ts` Step 5 前插入长度检测和路径切换

### Step 3：两路都跑社会心理学

- 路径 A：如果 claude-sonnet-4-5 装得下，直接跑；装不下则记录"超出上下文，需要 claude-3-5-sonnet-200k"
- 路径 B：层级提炼，输出到 `/tmp/`

### Step 4：输出结果到仓库

两个结果文件放到 `test-outputs/` 目录下供 review：
- `test-outputs/社会心理学-path-a-claude.md`（或失败记录）
- `test-outputs/社会心理学-path-b-hierarchical.md`

---

## 不在范围内

- 自动选择最优路径（先实现，再优化）
- 流式输出进度（中间 pass 可能需要几分钟）
- 缓存中间结果（第一版不做）
