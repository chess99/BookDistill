# Position Sizer — 仓位计算器（含期望值框架）

## 两个互补的框架

**埃尔德框架**（每笔交易的风险控制）：
- 2% 原则：单笔最大风险 = 账户的 2%
- 6% 原则：月度累计风险上限 = 账户的 6%

**撒普框架**（系统层面的期望值）：
- R 倍数：每笔交易的盈亏以"初始风险R"为单位衡量
- 期望值：系统长期每1R风险的平均回报
- 核心洞见：胜率不重要，期望值才重要

---

## 一、单笔仓位计算（埃尔德2%原则）

### 公式

```
初始风险 R = 入场价 - 止损价（做多）
最大持仓量 = (账户规模 × 2%) ÷ R
```

### 示例

- 账户：100,000 元
- 单笔最大风险：2,000 元（2%）
- 入场价：50 元，止损价：48 元 → R = 2 元
- 最大仓位：2,000 ÷ 2 = **1,000 股**

### 止损设置原则

- 止损距离至少 **1倍ATR**（平均真实波幅），避免被噪音触发
- **避开整数价格**（改为整数上下几个最小单位）
- 做多止损设在支撑位下方，做空止损设在阻力位上方

---

## 二、月度风险检查（埃尔德6%原则）

### 公式

```
本月已亏损 + 所有持仓风险敞口 ≤ 账户月初资金 × 6%
```

每个持仓的风险敞口 = (入场价 - 当前止损价) × 持仓量
（止损已移至盈亏平衡点以上的持仓，风险敞口为 0）

### 达到6%上限时

- 本月停止开新仓
- 可以管理现有持仓（调整止损、部分止盈）
- 等待下月重置，或等持仓止损上移释放额度

---

## 三、R倍数与期望值（撒普框架）

### R倍数

每笔交易的结果用"初始风险R"来衡量：

```
R倍数 = 实际盈亏 ÷ 初始风险R

示例：
- 初始风险 R = 500元
- 盈利 1500元 → +3R
- 亏损 500元 → -1R（止损出局）
- 亏损 1000元 → -2R（超出止损，滑点或跳空）
```

### 期望值公式

```
期望值 = (胜率 × 平均盈利R) - (败率 × 平均亏损R)

示例（胜率40%，平均盈利3R，平均亏损1R）：
期望值 = 0.4 × 3 - 0.6 × 1 = 1.2 - 0.6 = +0.6R
→ 每笔交易平均盈利 0.6R，系统有效
```

**关键洞见：**
- 期望值 > 0 = 系统长期盈利
- 期望值 < 0 = 无论多努力都会亏损
- 胜率 40% 但期望值 +0.6R，优于胜率 70% 但期望值 -0.1R

### 系统健康度评估

收集至少 30 笔历史交易，计算：

| 指标 | 计算方式 | 健康标准 |
|------|---------|---------|
| 期望值 | 平均R倍数 | > 0 |
| 最大连续亏损 | 历史最长亏损串 | 做好心理准备 |
| 盈亏比 | 平均盈利R ÷ 平均亏损R | > 1.5:1 |
| 机会频率 | 每月交易次数 | 越多越好（在质量保证下） |

## 五、特殊情况处理

**6%额度已满：**
```
⛔ 本月可用风险额度已耗尽
建议：暂停开新仓，管理现有持仓
如有持仓止损上移至盈亏平衡，会释放额度
```

**止损太近（< 1倍ATR）：**
```
⚠️ 止损距离小于1倍ATR，容易被噪音触发
建议将止损移至 [ATR止损价]，调整后仓位：[N] 股
```

**风险收益比不足2:1：**
```
⚠️ 风险收益比 [X:1]，低于建议的2:1
建议：寻找更好的入场点，或等待更好的机会
```

**期望值评估（如用户提供历史数据）：**
```
系统期望值：+[X]R（每笔交易平均盈利 [X] 倍初始风险）
评估：[系统有效 / 需要改进 / 系统失效]
建议：[具体改进方向]
```
